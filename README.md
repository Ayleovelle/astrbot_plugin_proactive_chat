# ⚠️ 实验性分支：模块化架构验证 (PoC)

> **注意：此分支包含不可运行的实验性代码，仅作存档和参考用途。**

## 📂 分支简介

这个分支 (`feature/modular-design-poc`) 记录了我们尝试将插件从单文件架构重构为模块化架构的一次探索。

**目前该分支已废弃且严重落后于主分支的进度，不会再有合并计划，相关代码仅做为早期探索的存档。**

新的模块化重构将于分支 dev/1.2.0 中进行。

## 🎯 目标

我们尝试解决 `main.py` 单文件过于臃肿的问题，并为后续开发"战未来"，计划将代码拆分为以下模块：

- `core/`: 核心逻辑（配置管理、数据管理、调度器、会话管理、TTS管理）
- `features/`: 功能模块（消息处理、自动触发、沉默计时器）
- `llm/`: LLM客户端
- `utils/`: 工具函数（时间工具、日志器）

## 🛑 暂停原因

在重构过程中，我们遇到了事件传播机制问题。具体表现为：

1. **事件传播问题**：模块化后的插件，Bot无法回复正常情况下的用户消息
2. **框架行为差异**：单文件架构的事件处理行为与模块化架构存在差异
3. **兼容性问题**：修复尝试未能完全解决私聊消息回复失败的问题

为了不影响插件的后续开发，我们决定暂时搁置这个方案。
